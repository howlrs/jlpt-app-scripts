# 絶対遵守ルール (最優先・例外なし)
# ═══════════════════════════════════════════════

以下の 10 条は **生成前後に必ず自己確認**し、違反する問題は出力せず修正すること。

1. **正解は一意** — 正解以外の 3 つが「明確な根拠で不正解」と説明可能なこと。「〜から」vs「〜ので」のような両方成立する選択肢は禁止。
2. **本文と正解の一貫性** — 読解/聴解で本文 (prerequisites) に書かれていない要素を正解にしない。正解は必ず本文から直接導出できること。
3. **選択肢は語句テキスト** — 並び替え問題 (cat=9) でも select_answer の value は **実際の語句** を入れる。数字「1」「2」「3」「4」だけを value にしない。
4. **漢字読み問題 (cat=2) は漢字必須** — sub.sentence に読み方を問う漢字が含まれていること。「わたしの（　　）は…」のように漢字が欠落している出題は禁止。
5. **穴埋め形式は「（　　）」に統一** — 全角括弧+全角スペース2個。「（）」「（ ）」「（　）」「＿＿＿」「星印★」 (並び替え以外) は禁止。
6. **HTMLタグ禁止** — `<u>`, `<b>`, `<i>`, `<span>` 等を出力に含めない。フロント側ではタグが素のテキストとして表示される。下線を引きたい場合は【】で囲む。
7. **レベル範囲を厳守** — N5/N4 で「〜てあります」「〜ているところです」「〜ことにします」等の中級複合文法を出さない。選択肢最長 12 字以内を目安。
8. **選択肢の品詞を統一** — い形容詞とな形容詞、動詞と名詞などが1問内で混在しない。
9. **選択肢の文字数は揃える** — 4 つの選択肢の文字数差を 5 字以内に収める (読解系を除く)。
10. **JSON 形式を厳守** — マークダウン記法禁止。`sub_questions` 配列、`select_answer` の `{key, value}` 構造、`answer` の key 番号文字列を必ず守る。

---

**出力命令**
日本語能力試験（JLPT）の**LEVEL**レベルの問題を作成します。この指示書は公式試験の形式・難易度・出題傾向を完全に再現するための厳格な指針です。各レベルの言語知識（文字・語彙・文法）および言語運用（読解・聴解）の特性を踏まえて作問してください。1リクエストあたり10題以上出力してください。

**レベル別特性**
- N1: 幅広い場面で使われる高度な日本語を理解できる（上級）
- N2: 日常的な場面で使われる日本語をある程度理解できる（中上級）
- N3: 日常的な場面で使われる日本語をある程度理解できる（中級）
- N4: 基本的な日本語を理解できる（初中級）
- N5: 基本的な日本語をある程度理解できる（初級）

**試験構成と問題タイプ**
1. 言語知識（文字・語彙・文法）
   - 漢字読み/書き
   - 語彙（意味合い/用法/文脈判断）
   - 文法（文の組み立て/文法形式の識別）
   - 文脈規定（文の並べ替え）

2. 読解
   - 情報検索（文章から特定情報を見つける）
   - 内容理解（短文/中文/長文）
   - 統合理解（複合的な情報処理・論理展開）
   - 主旨理解（文章全体の趣旨把握）

3. 聴解
   - 即時応答（短い会話に対する応答選択）
   - 課題理解（情報処理・設問対応）
   - 要点理解（会話・説明文の重要点把握）
   - 概要理解（まとまりのある会話・説明の全体像把握）

**問題階層構造**
- 大問（i.）: 問題セクションの基盤。出題指示・状況設定を含む（例：「次の文章を読んで、問いに答えなさい」）
- 中問/小問（i-j.）: 具体的な設問。選択肢や解答を含む。
- 選択肢は各行に1つずつ記載し、4つの選択肢があれば4行の出力とする。

**出力形式**
```json
[
  {
    "level_name": "N1～N5のいずれか",
    "category_id": "問題カテゴリ名と対応するIDを定義を参照",
    "category_name": "問題カテゴリ名と対応するIDを定義を参照",
    "sentence": "問題の指示文や状況説明（例：「次の文の（　）に入る最も適当なものを、1・2・3・4から一つ選びなさい」）",
    "prerequisites": "読解・聴解の場合の前提文章または会話文",
    "sub_questions": [
      {
        "sentence": "具体的な問題文（例：「彼女は毎日（　）8時に起きます」）",
        "prerequisites": "並べ替え問題などの前提文章",
        "select_answer": [
          {"key": "1", "value": "選択肢1の内容"},
          {"key": "2", "value": "選択肢2の内容"},
          {"key": "3", "value": "選択肢3の内容"},
          {"key": "4", "value": "選択肢4の内容"}
        ],

      }
    ]
  }

```

**問題作成における厳格基準**
1. レベル適合性: 各レベルの言語項目・難易度に厳密に準拠
   - N1: 慣用句・四字熟語・擬態語・擬声語、複雑な複合文、評論文・論説文理解
   - N2: 日常語彙1,000〜6,000語、条件節・理由節などの複文、一般的社会問題文の理解
   - N3: 日常語彙3,000語程度、基本的な複文構造、平易な説明文・物語文の理解
   - N4: 基本語彙1,500語程度、初級後半の文法、生活場面の短文理解
   - N5: 基本語彙800語程度、初級前半の文法、簡単な文の理解

2. 出題精度
   - 漢字問題: レベル別漢字リストに準拠し、使用頻度・複雑さを考慮
   - 語彙問題: 使用場面・ニュアンスの違いを明確に区別できるよう設計
   - 文法問題: 各文法項目の用法制限・例外を正確に反映
   - 読解問題: 文章構造・論理展開・含意を考慮した設問設計
   - 聴解問題: 実際の会話速度・自然な間・音声特性を想定した設問

3. 問題構造
   - 選択肢: 4択を基本とし、紛らわしく適切な誤答選択肢を用意
   - 穴埋め問題: 代入箇所と選択肢数の整合性を厳密に確認
   - 文脈問題: 文章全体の流れに対する理解を正確に測定
   - 情報抽出問題: 複数情報から特定要素を抽出する能力を測定

4. 内容倫理
   - 文化的背景: 多様な文化・背景を考慮した中立的内容
   - センシティブ項目: 政治・宗教・性・人種・差別等のセンシティブな内容の排除
   - 時事問題: 一般常識範囲内の無難な時事的内容に限定

**技術的注意事項**
- 数字は半角、括弧類は全角（）【】を使用
- 穴埋め箇所は（　　）で統一表記
- 必ず有効なJSONとして出力すること
- sub_questionsの各項目では、select_answerとanswerは必須項目
- 回答選択肢は必ずselect_answerに格納し、sentenceには含めない
- answerには正解となる選択肢のkeyを必ず含める
- 大問のsentenceには問題指示を、小問のsentenceには具体的な問題文を入れる
- 途中で出力が切れないよう、問題数を調整する（10題程度を推奨）

**正解の根拠と誤答の設計**
各問題を作成する際、以下を内部的に確認してから出力すること（出力には含めない）:
1. 正解が唯一無二であること（複数正解が成立しないこと）
2. 正解の根拠を明確に説明できること
3. 各誤答が「一見正しそうだが、明確な理由で不正解」であること
4. 誤答の種類を多様化すること（意味のずれ、文法的不適合、文脈不一致、レベル不適切など）

**選択肢の品質基準**
- 4つの選択肢は長さ・構造・語彙レベルを揃えること
- 消去法だけで即答できる明らかに不自然な選択肢を含めないこと
- 全ての選択肢が問題の文脈に関連していること
- 正解の位置（1〜4）を偏りなく分散させること

**良問の例（N3語彙問題）**
```json
{
  "level_name": "n3",
  "category_id": "3",
  "category_name": "語彙-文脈規定",
  "sentence": "次の文の（　　）に入る最も適当なものを、1・2・3・4から一つ選びなさい。",
  "prerequisites": "",
  "sub_questions": [
    {
      "sentence": "この薬は熱を（　　）効果があります。",
      "prerequisites": "",
      "select_answer": [
        {"key": "1", "value": "下げる"},
        {"key": "2", "value": "落とす"},
        {"key": "3", "value": "引く"},
        {"key": "4", "value": "冷やす"}
      ],
      "answer": "1"
    }
  ]
}
```
- 正解「下げる」: 「熱を下げる」は自然なコロケーション
- 誤答「落とす」: 温度ではなく物理的に落とすニュアンス
- 誤答「引く」: 「風邪を引く」の混同を狙う
- 誤答「冷やす」: 体を冷やすと混同しやすいが、熱には不適切

**悪問の例 (絶対に生成しないこと)**

```
NG例1: 正解が複数成立する
  問題: 「彼は（　　）働いている。」
  選択肢: 1.一生懸命  2.熱心に  3.真面目に  4.机で
  → 1/2/3 全て正解になりうる
  Fix: 意味の軸を変える
  選択肢: 1.一生懸命  2.ゆっくり  3.楽しく  4.静かに

NG例2: 消去法で即答できる (品詞不一致)
  問題: 「毎日（　　）を食べます。」
  選択肢: 1.食べる  2.走る  3.泳ぐ  4.ごはん → 品詞違い
  Fix: 全て名詞で揃える
  選択肢: 1.ごはん  2.コーヒー  3.おちゃ  4.パン

NG例3: レベル不適切
  N5問題: 「この道は（　　）を凌駕している。」
  → N5 に N1 語彙「凌駕」混入
  Fix: N5 範囲の語彙で作問
  N5問題: 「この道は（　　）広いです。」1.とても 2.もう 3.まだ 4.よく

NG例4: 選択肢の長さが不均一
  1.はい  2.いいえ  3.そのような状況においては判断を保留すべきだ  4.たぶん
  Fix: 文字数を揃える
  1.賛成です  2.反対です  3.保留します  4.分かりません

NG例5: 正解が本文に書かれていない (読解)
  本文: 「田中さんは日曜日に本を読みました。」
  問題: 「田中さんは日曜日に何をしましたか。」
  正解: 「映画を見た」← 本文にない！
  Fix: 正解は必ず本文から導出
  正解: 「本を読んだ」

NG例6: 並び替え問題で value が数字のみ (致命的)
  sub.sentence: "きょうは　＿＿＿　＿＿＿　★　＿＿＿　ます"
  select_answer: [{"key":"1","value":"1"},{"key":"2","value":"2"},{"key":"3","value":"3"},{"key":"4","value":"4"}]
  → 並べるべき語句が不明、解答不能
  Fix: value に語句を入れる
  select_answer: [{"key":"1","value":"がっこうに"},{"key":"2","value":"じてんしゃで"},{"key":"3","value":"いき"},{"key":"4","value":"を"}]
  answer: "3" (星位置の語)

NG例7: 漢字読み問題で漢字なし (致命的)
  問題: "わたしの（　　）は きょうとです。"  ← 漢字がどこにもない
  選択肢: [なまえ, まち, くに, うち]
  → どれを読ませたいか不明
  Fix: 漢字を sub.sentence に入れる
  問題: "わたしの（国）は きょうとです。"
  選択肢: [なまえ, まち, くに, うち]  正解: くに

NG例8: 並び替え問題で正解並び順が文として成立しない (致命的)
  sub.sentence: "あそこに　（　　）　（　　）　★　（　　）　います。"
  select_answer: [{"key":"1","value":"の"},{"key":"2","value":"いっぴき"},{"key":"3","value":"ねこが"},{"key":"4","value":"ちいさい"}]
  正解 key="1" → 「あそこに ねこが の ちいさい いっぴき います」 ← 文法破綻
  Fix: 並べ直して文が成立する位置に★を置く
  正解の並び: ちいさい→ねこが→が→いっぴき、★位置は「が」の key

NG例9: HTMLタグ混入
  問題: "わたしは まいにち <u>いそがしい</u>です。" ← タグが素のテキストに
  Fix: タグを使わず自然文で
  問題: "わたしは まいにち いそがしいです。" (強調不要、または【】で)

NG例10: 形式揺れ
  問題: "コーヒーを（　）のみます。" ← スペース1個 (NG)
  Fix: 全角スペース2個に統一
  問題: "コーヒーを（　　）のみます。"
```

**品質検証項目 (出力前の自己確認チェックリスト)**

### コア
- [ ] 正解が唯一無二 (他3つが明確に不正解と説明可能)
- [ ] 正解が本文・文脈から直接導出できる (読解系)
- [ ] select_answer と answer の必須入力
- [ ] 有効なJSON形式 (マークダウン記法なし)
- [ ] 全選択肢が4つ揃っている
- [ ] 選択肢の長さ・構造・品詞・語彙レベルが均一

### カテゴリ固有
- [ ] 漢字読み (cat=2): sub.sentence に読み方を問う漢字が含まれている
- [ ] 並び替え (cat=9): select_answer.value に実際の語句が入っており、正解 key で並べた文が成立する
- [ ] 読解系 (cat=10-17): parent.prerequisites に本文が含まれ、正解が本文から導出可能

### 書式
- [ ] 穴埋めは「（　　）」(全角括弧+全角スペース2個)
- [ ] HTMLタグ (<u> 等) を出力に含めない
- [ ] レベル範囲を厳守 (N5/N4 に中級複合文法なし)
- [ ] 消去法だけで即答できる問題なし

### メタ
- [ ] 正解の位置 (1〜4) を偏りなく分散
- [ ] 内容の中立性・適切性
- [ ] 誤字・脱字・文法エラーなし

以上すべてにチェックが入らない問題は**出力から除外**し、別の問題を追加生成すること。
