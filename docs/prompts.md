# jlpt-app-scripts プロンプト設計

## プロンプト構成

問題生成時に以下のプロンプトを組み合わせてGemini APIに送信する。

```
[create-question_to_json.md]  ── 生成指示・出力フォーマット
        +
[base-info.md]                ── JLPTの基本情報・ガイドライン
        +
[{level}/ja-question.md]      ── レベル別の語彙・文法制約
```

## プロンプトファイル一覧

### 共通プロンプト

| ファイル | サイズ | 内容 |
|---------|--------|------|
| `base-info.md` | 11 KB | JLPT試験構成、配点、評価基準、品質チェックリスト |
| `create-question_to_json.md` | 5.8 KB | JSON出力フォーマット指定、問題分類体系、品質基準 |

### レベル別プロンプト

| レベル | ファイル数 | 内容 |
|--------|----------|------|
| N1（上級） | 5 | 語彙(10,000語)、複雑文法、抽象テキスト、専門内容 |
| N2 | 2 | 問題文・カテゴリ定義 |
| N3（中級） | 2 | 問題文・カテゴリ定義 |
| N4 | 5 | 基礎語彙・文法、日常場面 |
| N5（初級） | 5 | 語彙(800語)、基本文法、具体的場面 |

### 各レベルのファイル構成

完全版（N1, N4, N5）：

| ファイル | 内容 |
|---------|------|
| `ja-question.md` | 語彙範囲・文法範囲・出題傾向 |
| `ja-categories.md` | カテゴリ定義（N1: 22カテゴリ, N5: 19カテゴリ） |
| `ja-answer.md` | 正解の生成ガイダンス |
| `ja-hint.md` | ヒント生成ガイダンス |
| `ja-select.md` | 選択肢生成ガイダンス |

## 品質管理

`base-info.md` に定義された12項目の品質チェックリスト：

1. レベル適正性
2. 問題文の明確さ
3. 選択肢の妥当性
4. 正解の正確さ
5. 文化的バイアスの排除
6. 倫理的配慮
7. 公式フォーマット準拠
8. 半角数字使用
9. 全角括弧使用
10. 有効なJSON出力
11. レベル別語彙・文法制約の遵守
12. カテゴリ分類の正確さ

## 問題分類体系

3大カテゴリ：

| カテゴリ | 内容 |
|---------|------|
| 言語知識 | 語彙・文法・読解 |
| 読解 | 短文・中文・長文・情報検索 |
| 聴解 | 課題理解・ポイント理解・概要理解・即時応答 |
