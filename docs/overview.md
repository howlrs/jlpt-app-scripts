# jlpt-app-scripts 概要

## プロジェクト情報

| 項目 | 内容 |
|------|------|
| 言語 | Rust (Edition 2024) |
| AI連携 | Google Gemini API |
| データベース | Google Firestore |
| 対象レベル | N1〜N5（全5レベル） |

## 概要

JLPT問題の大規模自動生成・加工・データベース投入を行うデータパイプライン。Google Gemini APIでカテゴリベースのAI生成された問題を、段階的に検証・類似排除・加工し、Firestoreに格納する。全レベル合計で18,060件のsub_questionsを生成。

## 責務

1. AI（Gemini）によるカテゴリベースの大量問題生成（レベル×カテゴリごとに生成）
2. JSON→構造体パース・バリデーション
3. 類似排除（Levenshtein距離、85%閾値）
4. ID採番（UUID）
5. レベルID正規化
6. カテゴリメタデータ抽出
7. Firestoreへのバッチ投入
8. 既存DBデータの確認・置換

## スキーマ概要

生成された問題には `generated_by` フィールドが含まれ、生成に使用したGeminiモデル名を記録する。

## japanese-app との違い

| 項目 | japanese-app | jlpt-app-scripts |
|------|-------------|------------------|
| 対象レベル | N2, N3 | N1〜N5（全レベル） |
| 生成方式 | ランダム生成 | カテゴリベース生成 |
| 重複排除 | 完全一致 | 類似排除（Levenshtein距離85%閾値） |
| パイプライン | 生成→結合→パース | 生成→パース→バリデーション→類似排除→採番→レベリング→カテゴリ化→DB投入 |
| DB連携 | なし | Firestore投入・確認・置換あり |
| プロンプト | 2レベル分 | 5レベル分（完全版） |
| チェックポイント | なし | チェックポイント/リジューム対応 |
